# Best-Practices Matrix

| Technology | Prefer | Avoid | Verify |
|---|---|---|---|
| .NET / ASP.NET Core | Minimal APIs or controllers with thin endpoint logic, DI, typed options, async I/O, structured logging | Blocking calls, service locator, fat controllers, leaking exceptions | `dotnet build`, `dotnet test`, `dotnet format` |
| EF Core | Short-lived DbContext, no-tracking reads, explicit includes/projections, migrations, query plans for hot paths | N+1 queries, lazy-loading surprises, client-side filtering, uncontrolled tracking | Integration tests, generated SQL review |
| Python | `src/` layout, typed public functions, explicit dependencies, isolated environment | Global mutable state, hidden imports, untyped service boundaries | Ruff, Pyright, pytest |
| FastAPI | Pydantic models at boundaries, dependency injection, async only for real async I/O, lifespan for startup/shutdown | Blocking work in async endpoints, returning ORM objects directly, hidden auth | OpenAPI check, API tests |
| SQLAlchemy 2.x | `select()` style, session per unit of work, explicit transaction scope, eager loading/projections | Legacy Query style in new code, session globals, N+1, implicit commits | Tests with real DB where possible |
| Alembic | Small deterministic migrations, reviewed autogenerated diffs, safe data migration plan | Editing applied migrations, destructive changes without plan, multi-head drift | `alembic heads`, upgrade/downgrade smoke test |
| uv | Commit `uv.lock`, use `uv sync --locked` in CI, use `uv run` for tools | Manually editing lockfiles, mixing pip/poetry ad hoc in same project | `uv sync --locked` |
| Ruff | Central `pyproject.toml`, format + lint in CI, clear per-rule exceptions | Broad `ignore = [...]`, disabling rules locally without reason | `ruff format --check`, `ruff check` |
| Pyright | Strict mode for application code, explicit public types | `Any` leakage, blanket ignores | `pyright` |
| TypeScript | `strict`, discriminated unions, typed API clients, no implicit any | `any`, unsafe casts, unchecked JSON | `tsc --noEmit` or project script |
| React | Function components, hooks rules, derived state, server-state library when needed | Effects for pure computation, mutable state, large monolithic components | unit/component tests, build |
| Vue | Composition API, typed props/emits, composables for reusable logic, Pinia for app state | Excessive watchers, deep mutable shared state, business logic in templates | `vue-tsc`, unit/component tests |
| Vite | Env variables with `VITE_` prefix for client exposure, separate typecheck, optimized build config | Assuming Vite transpilation means type safety, leaking server secrets to client | `npm run typecheck`, `npm run build` |
| PostgreSQL | Proper indexes, `EXPLAIN`, constraints, transactions, JSONB only when justified | Over-indexing, missing constraints, unbounded queries, `SELECT *` hot paths | query plans, integration tests |
| SQL Server | Indexes based on workload, execution plans, parameterized queries, migrations | NOLOCK as default, non-sargable filters, over-indexing | actual execution plans, integration tests |
| Docker Compose | Healthchecks, named volumes, explicit ports, non-root containers when possible | Baking secrets into images, `latest` tags in production, mutable local state assumptions | `docker compose config`, smoke tests |
